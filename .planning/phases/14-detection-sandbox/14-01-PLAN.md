<objective>
Fix tmux set-environment for Docker sandbox sessions by pre-generating session IDs on the host and calling SetEnvironment from Go instead of embedding the call inside docker exec shell strings.

Purpose: tmux set-environment inside a Docker container cannot reach the host tmux server socket at /tmp/tmux-<uid>/default. This breaks session ID tracking for sandbox sessions, making GetSessionIDFromTmux() return empty and preventing resume/restart from working correctly.

Output: Modified command builders in instance.go that conditionally skip in-shell tmux set-environment for sandbox sessions, plus host-side SetEnvironment calls. Comprehensive tests verifying sandbox vs non-sandbox behavior.
</objective>

**Call sites to fix (all in instance.go):**

// For sandbox sessions, set tool session IDs from Go (host-side) since
// in-shell tmux set-environment cannot reach host tmux from inside Docker.
if i.IsSandboxed() {
    if i.pendingClaudeSessionID != "" {
        _ = i.tmuxSession.SetEnvironment("CLAUDE_SESSION_ID", i.pendingClaudeSessionID)
        i.ClaudeSessionID = i.pendingClaudeSessionID
        i.pendingClaudeSessionID = ""
    }
}

**Implementation pattern for the conditional:** Use a helper function to keep code DRY:
```go
// tmuxSetEnvPrefix returns the "tmux set-environment KEY VALUE; " shell prefix
// for non-sandbox sessions, or empty string for sandbox sessions (where
// SetEnvironment is called from Go on the host side).
func (i *Instance) tmuxSetEnvPrefix(key, value string) string {
    if i.IsSandboxed() {
        return ""
    }
    return fmt.Sprintf("tmux set-environment %s %s; ", key, value)
}
```
